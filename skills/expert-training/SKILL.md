# Expert Training Skill v4.0 — 专家库训练系统

> 用于训练、增量优化、质量检验圆桌讨论专家档案。
> 适用于 book-distillation 系统的 Phase 1.5 圆桌深化 Agent。

## 触发词

- "训练新专家"、"续训专家"、"增量训练"
- "提升专家质量"、"优化专家档案"
- "专家名 + 训练/续训/提升"
- "批量优化"、"L3测试"、"质量监控"
- "解决问题"、"升级skills"

---

## 核心改进（v4.0 vs v3.0）

| 维度 | v3.0 问题 | v4.0 改进 |
|:---|:---|:---|
| L3测试覆盖 | 仅手动执行，覆盖率低 | **自动化批量测试**，按类别分组执行 |
| 低分专家处理 | 单个手动优化 | **批量优化引擎**，并行处理多位专家 |
| 知乎增强搜索 | 手动执行，进度滞后 | **自动化采集**，批量执行并整合 |
| 质量监控 | 无实时报告 | **质量仪表盘**，实时统计和预警 |
| 进度追踪 | 手动更新training-progress.md | **自动同步**，每次操作自动更新 |

## v4.0 核心功能模块

### 模块1：自动化L3测试系统

**目的**：确保所有专家都通过去同质化测试，避免圆桌讨论中"说一样的话"。

**触发条件**：
- 新专家训练完成时自动触发
- 每周定时批量检查未测试专家
- 用户手动触发"L3测试"

**执行流程**：
```markdown
1. **分组策略**
   - 按专家分类自动分组（business/psychology/philosophy等）
   - 每组3-4位专家进行正交性测试
   - 跨组测试（如：中文知识分子 vs 文学哲学）

2. **话题生成**
   - 自动生成3个测试话题（覆盖该组专家的核心领域）
   - 话题格式："{领域}视角下的{热点问题}"

3. **批量模拟发言**
   - 每位专家对3个话题分别发言
   - 发言长度：200-300字
   - 保持专家独特风格

4. **正交性评估**
   | 维度 | 25分 | 评估标准 |
   |:---|:---|:---|
   | 立场差异 | 25 | 专家立场是否有明显分歧 |
   | 论证差异 | 25 | 论证方式是否各具特色 |
   | 关注点差异 | 25 | 是否从不同角度切入 |
   | 语言风格差异 | 25 | 语言风格是否可区分 |

5. **质量门**
   - ≥75分：通过
   - <75分：触发增量训练，调整专家档案以增加区分度
```

**自动化规则**：
- L3测试通过率 < 80% 时自动预警
- 未通过专家自动加入优化队列
- 测试结果自动更新到training-progress.md

---

### 模块2：批量优化引擎

**目的**：自动识别低分专家并批量优化，提升整体质量。

**触发条件**：
- 专家评分 < 93分时自动触发
- 用户手动触发"批量优化"
- 每日定时检查低分专家

**执行流程**：
```markdown
1. **低分专家识别**
   - 扫描所有专家档案
   - 识别评分 < 93分的专家
   - 按评分排序，优先处理最低分

2. **问题诊断**
   - 读取每位低分专家的档案
   - 识别薄弱环节（理论深度、批评、模拟发言等）
   - 生成优化建议

3. **并行优化**
   - 同时处理3-5位专家
   - 针对性补充内容
   - 重新评分

4. **质量门**
   - 优化后评分 ≥ 93分：通过
   - 优化后评分 < 93分：进入下一轮优化
   - 最多3轮优化，仍不达标则标注[质量待提升]
```

**批量优化策略**：
| 问题类型 | 优化操作 |
|:---|:---|
| 理论深度不足 | 补充搜索 → 增加案例和来源 |
| 批评不足 | 搜索"{专家名} 批评"、"{专家名} 争议" |
| 模拟发言差 | 重写模拟发言，强化风格特征 |
| 圆桌角色卡缺失 | 补充完整角色卡 |
| 语录不足 | 扩展至20条以上，含来源验证 |

---

### 模块3：知乎增强搜索自动化

**目的**：批量完成专家的深度内容采集，提升档案质量。

**触发条件**：
- 专家档案创建时自动触发
- 用户手动触发"知乎增强"
- 每周定时检查未完成专家

**执行流程**：
```markdown
1. **搜索策略**
   - 关键词："{专家名} 核心思想"、"{专家名} 评价"、"{专家名} 语录"
   - 每位专家最多3次搜索
   - 频率控制：每分钟最多5次请求

2. **内容整合**
   - 提取深度讨论和批评视角
   - 整合到档案的"批评与争议"章节
   - 补充语录和案例

3. **进度追踪**
   - 记录已完成的专家
   - 更新training-progress.md
   - 未完成专家自动加入队列
```

**自动化规则**：
- 知乎增强完成率 < 80% 时自动预警
- 未完成专家自动加入采集队列
- 采集结果自动整合到档案

---

### 模块4：质量监控仪表盘

**目的**：实时监控专家库质量状态，自动预警和追踪。

**监控指标**：
```markdown
| 指标 | 计算方式 | 预警阈值 |
|:---|:---|:---|
| 平均评分 | 所有专家评分总和 / 专家数量 | < 94分 |
| 95分以上占比 | 95分以上专家数 / 总专家数 | < 80% |
| L3测试通过率 | L3通过专家数 / 总专家数 | < 80% |
| 知乎增强完成率 | 知乎完成专家数 / 总专家数 | < 80% |
| 低分专家数 | 评分 < 93分的专家数 | > 5位 |
```

**自动报告**：
- 每次操作后自动更新training-progress.md
- 生成质量报告（JSON格式）
- 预警通知（当指标低于阈值时）

## 训练流程（v4.0 升级版）

### Step 1：专家画像初始化

**输入**：专家名（中英文）、领域分类

**操作**：
1. 确定专家分类（business/psychology/philosophy/technology/history/self-help/economics/science/cross-domain/chinese-intellectuals）
2. 选择对应模板变体（通用/科学家/哲学家/跨界专家/中文知识分子）
3. 确定文件路径：`experts/{category}/{expert-slug}.md`

**质量门**：确认该专家不在已有库中（避免重复）

---

### Step 2：多源深度搜索

**搜索策略**（按优先级）：

| 优先级 | 来源类型 | 示例 |
|:---|:---|:---|
| P0 | 学术百科 | Stanford Encyclopedia of Philosophy, Britannica, 维基百科 |
| P1 | 原始著作 | 豆瓣读书、Google Scholar、JSTOR |
| P2 | 权威媒体 | 环球人物、知识分子、Nature/Science |
| P3 | 中文百科 | 百度百科（仅作补充，不作主力） |
| P4 | 知乎讨论 | 使用 zhihu-mcp 搜索该专家的深度讨论 |

**搜索清单**（至少完成）：
- [ ] 专家生平与历史地位
- [ ] 核心理论/观点（≥5 个）
- [ ] 经典著作（≥4 部，含出版年份）
- [ ] 经典语录（≥20 条，含出处+验证）
- [ ] 批评与争议（≥5 条，含批评者身份）
- [ ] 与其他专家的思想关系（≥3 位）
- [ ] 发言风格特征（语气/句式/论证方式）
- [ ] 圆桌角色卡（角色定位、核心武器、发言节奏、典型质疑、盟友/对手/互补者）

**知乎增强搜索**（v4.0自动化）：
```
使用 zhihu-mcp 的 search_content 工具：
- 搜索关键词："{专家名} 核心思想"、"{专家名} 评价"、"{专家名} 语录"
- 目的：获取中文社区的深度讨论和批评视角
- 限制：每个专家最多 3 次搜索，避免频率过高
- 自动化：专家创建时自动触发，批量处理
```

---

### Step 3：结构化档案生成

**模板选择**：
- 哲学家 → `expert-profile-philosophy.md`
- 科学家 → `expert-profile-science.md`
- 跨界专家 → `expert-profile-cross-domain.md`
- 中文知识分子 → `expert-profile-chinese-intellectuals.md`
- 其他 → `expert-profile-template.md`

**强制章节**（全部必须填写，不可省略）：

| 章节 | 最低要求 | 质量标准 |
|:---|:---|:---|
| 基本信息 | 7 项属性 | 含 MBTI [推测] 标注 |
| 核心立场 | 1 句话 | 有锋芒，体现独特性 |
| 经典著作 | ≥4 部 | 含年份、核心贡献、URL 链接 |
| 核心理论 | ≥5 个 | 每个含：关键概念 + 深层逻辑 + 案例 + 来源 |
| 历史语境 | 5 维表格 | 政治/经济/思想/技术/社会 |
| 经典语录 | ≥20 条 | 含出处 + 验证状态 |
| 发言风格 | 4 维描述 | 语气/句式/论证/标志性表达 |
| 模拟发言 | ≥4 个场景 | 含：独立阐述 + 回应质疑 + 跨专家对话 + 总结陈词 |
| 批评与争议 | ≥5 条 | 每条含：批评者 + 内容 + 第一人称回应 |
| 圆桌角色卡 | 完整 | 角色定位+核心武器+发言节奏+典型质疑+盟友/对手/互补者 |
| 搜索来源 | ≥6 个 URL | 优先学术来源 |

---

### Step 4：L1 质量自检

**评分维度**（7 维度 × 20 分 = 满分 140）：

| 维度 | 20 分标准 | 扣分项 |
|:---|:---|:---|
| 信息完整度 | 所有章节完整，无遗漏 | 缺 1 项扣 2 分 |
| 来源可靠性 | ≥6 个权威来源，含学术来源 | 仅百度百科扣 5 分，缺 URL 扣 3 分 |
| 理论深度 | 5 个理论均有案例+来源 | 缺案例扣 2 分/个，缺来源扣 1 分/个 |
| 批评平衡性 | ≥5 条批评，含第一人称回应 | 缺回应扣 3 分/条，<5 条扣 5 分 |
| 跨领域覆盖 | 覆盖 ≥3 个相关领域 | 仅本领域扣 8 分 |
| 语录准确性 | ≥20 条，均有出处+验证 | 缺出处扣 1 分/条，<20 条扣 2 分/条 |
| 模拟发言质量 | ≥4 场景，风格还原度高 | <4 场景扣 5 分，风格不鲜明扣 3 分 |

**质量门**：
- **≥110 分**：优秀，直接通过
- **90-109 分**：合格，可进入 L2
- **<90 分**：不合格，必须增量训练后再检

---

### Step 5：L2 仿真发言测试

**测试方法**：给定 3 个话题，模拟该专家在圆桌讨论中的发言。

**评分维度**（4 维度 × 25 分 = 满分 100）：

| 维度 | 25 分标准 |
|:---|:---|
| 风格还原度 | 语气、句式、论证方式与该专家高度一致 |
| 理论深度 | 发言体现该专家的核心理论框架 |
| 独特性 | 与其他专家有明显区分，不"撞车" |
| 锋芒与立场 | 有明确立场，不和稀泥，敢于质疑 |

**质量门**：
- **≥85 分**：优秀，通过
- **75-84 分**：合格，可使用
- **<75 分**：不合格，需增量训练

---

### Step 6：L3 跨专家去同质化（v4.0自动化）

**目的**：确保同类专家在圆桌讨论中不会"说一样的话"。

**v4.0自动化流程**：
1. **自动分组**：按专家分类自动分组（3-4位/组）
2. **话题生成**：自动生成3个测试话题
3. **批量模拟**：每位专家对3个话题分别发言
4. **正交性评估**：

| 维度 | 25 分 | 评估标准 |
|:---|:---|:---|
| 立场差异 | 25 | 3 位专家的立场是否有明显分歧 |
| 论证差异 | 25 | 论证方式是否各具特色（数据/案例/逻辑/直觉） |
| 关注点差异 | 25 | 是否从不同角度切入（宏观/微观/历史/未来） |
| 语言风格差异 | 25 | 语言风格是否可区分（格言/数据/故事/质问） |

**质量门**：总分 ≥75 分通过，<75 分需调整专家档案以增加区分度。

**自动化规则**：
- L3测试通过率 < 80% 时自动预警
- 未通过专家自动加入优化队列
- 测试结果自动更新到training-progress.md

---

### Step 7：增量训练（v4.0批量优化）

**触发条件**：
- L1 < 90 分
- L2 < 75 分
- L3 < 75 分
- 专家评分 < 93 分

**v4.0批量优化流程**：
1. **自动识别**：扫描所有专家，识别低分专家
2. **问题诊断**：读取档案，识别薄弱环节
3. **并行优化**：同时处理3-5位专家
4. **质量门**：优化后评分 ≥ 93分通过，否则进入下一轮

**增量策略**：

| 问题类型 | 增量操作 |
|:---|:---|
| 理论深度不足 | 补充搜索 → 增加案例和来源 |
| 批评不足 | 搜索 "{专家名} 批评"、"{专家名} 争议" → 补充批评条目 |
| 跨领域薄弱 | 搜索 "{专家名} 影响"、"{专家名} 跨领域" → 补充跨领域贡献 |
| 模拟发言差 | 重写模拟发言，强化风格特征 |
| 与其他专家撞车 | 调整核心立场措辞，增加独特角度 |
| 圆桌角色卡缺失 | 补充完整角色卡 |
| 语录不足 | 扩展至20条以上，含来源验证 |

**增量上限**：每个专家最多 3 轮增量训练，仍不达标则标注 `[质量待提升]`。

## 圆桌角色卡格式（v4.0增强版）

每个专家档案末尾增加"圆桌角色卡"，可直接用于讨论编排：

```markdown
## 圆桌角色卡

| 属性 | 内容 |
|:---|:---|
| **角色定位** | {如：价值诊断者、数据验证者、实践检验者} |
| **核心武器** | {该专家最独特的 1-2 个理论/方法} |
| **发言节奏** | {如：先听后说、主动出击、回应式发言} |
| **擅长环节** | {如：核心观点审视、盲点发现、跨领域连接} |
| **典型质疑** | {3-5 个该专家会提出的问题} |
| **盟友** | {观点相近的专家} |
| **对手** | {观点对立的专家} |
| **互补者** | {视角互补的专家} |
```

## 防幻觉规则（强制）

1. **事实必须有来源**：每个事实至少 1 个 URL 支撑
2. **无法验证则标注**：`[待核实]`
3. **MBTI 标注推测**：`[推测]（依据：...）`
4. **语录需出处**：无出处标注 `[出处待查]`
5. **来源 URL 化**：所有来源必须包含可点击链接
6. **学术优先**：SEP > Britannica > 维基百科 > 百度百科
7. **v4.0新增**：批量操作时必须逐个验证，不可跳过
