# Expert Training Skill v3.0 — 专家库训练系统

> 用于训练、增量优化、质量检验圆桌讨论专家档案。
> 适用于 book-distillation 系统的 Phase 1.5 圆桌深化 Agent。

## 触发词

- "训练新专家"、"续训专家"、"增量训练"
- "提升专家质量"、"优化专家档案"
- "专家名 + 训练/续训/提升"

## 核心改进（v3.0 vs v2.0）

| 维度 | v2.0 问题 | v3.0 改进 |
|:---|:---|:---|
| 质量方差 | 最低 78 分，最高 130 分 | 引入分级质量门，低于 90 分自动触发增量 |
| L3 去同质化 | 未实施 | 新增"正交检验"步骤 |
| 批评深度 | 部分专家仅 3 条 | 强制 ≥4 条批评，含"最强反对意见" |
| 来源权威性 | 依赖百度百科/CSDN | 优先学术来源（SEP/Stanford/原始论文） |
| 模拟发言 | 2-3 个场景 | 强制 ≥4 个场景，含"跨专家对话" |
| 圆桌适配 | 通用描述 | 增加"圆桌角色卡"——直接可用于讨论编排 |

## 训练流程（7 步法）

### Step 1：专家画像初始化

**输入**：专家名（中英文）、领域分类

**操作**：
1. 确定专家分类（business/psychology/philosophy/technology/history/self-help/economics/science/cross-domain）
2. 选择对应模板变体（通用/科学家/哲学家/跨界专家）
3. 确定文件路径：`experts/{category}/{expert-slug}.md`

**质量门**：确认该专家不在已有库中（避免重复）

---

### Step 2：多源深度搜索

**搜索策略**（按优先级）：

| 优先级 | 来源类型 | 示例 |
|:---|:---|:---|
| P0 | 学术百科 | Stanford Encyclopedia of Philosophy, Britannica, 维基百科 |
| P1 | 原始著作 | 豆瓣读书、Google Scholar、JSTOR |
| P2 | 权威媒体 | 环球人物、知识分子、Nature/Science |
| P3 | 中文百科 | 百度百科（仅作补充，不作主力） |
| P4 | 知乎讨论 | 使用 zhihu-mcp 搜索该专家的深度讨论 |

**搜索清单**（至少完成）：
- [ ] 专家生平与历史地位
- [ ] 核心理论/观点（≥5 个）
- [ ] 经典著作（≥4 部，含出版年份）
- [ ] 经典语录（≥8 条，含出处）
- [ ] 批评与争议（≥4 条，含批评者身份）
- [ ] 与其他专家的思想关系（≥3 位）
- [ ] 发言风格特征（语气/句式/论证方式）

**知乎增强搜索**（可选）：
```
使用 zhihu-mcp 的 search_content 工具：
- 搜索关键词："{专家名} 核心思想"、"{专家名} 评价"、"{专家名} 语录"
- 目的：获取中文社区的深度讨论和批评视角
- 限制：每个专家最多 3 次搜索，避免频率过高
```

---

### Step 3：结构化档案生成

**模板选择**：
- 哲学家 → `expert-profile-philosophy.md`
- 科学家 → `expert-profile-science.md`
- 跨界专家 → `expert-profile-cross-domain.md`
- 其他 → `expert-profile-template.md`

**强制章节**（全部必须填写，不可省略）：

| 章节 | 最低要求 | 质量标准 |
|:---|:---|:---|
| 基本信息 | 7 项属性 | 含 MBTI [推测] 标注 |
| 核心立场 | 1 句话 | 有锋芒，体现独特性 |
| 经典著作 | ≥4 部 | 含年份、核心贡献、URL 链接 |
| 核心理论 | ≥5 个 | 每个含：关键概念 + 深层逻辑 + 案例 + 来源 |
| 历史语境 | 5 维表格 | 政治/经济/思想/技术/社会 |
| 经典语录 | ≥8 条 | 含出处 + 验证状态 |
| 发言风格 | 4 维描述 | 语气/句式/论证/标志性表达 |
| 模拟发言 | ≥4 个场景 | 含：独立阐述 + 回应质疑 + 跨专家对话 + 总结陈词 |
| 批评与争议 | ≥4 条 | 每条含：批评者 + 内容 + 第一人称回应 |
| 圆桌角色卡 | 完整 | 擅长环节 + 质疑方向 + 与其他 ≥3 位专家的关系 |
| 搜索来源 | ≥6 个 URL | 优先学术来源 |

---

### Step 4：L1 质量自检

**评分维度**（7 维度 × 20 分 = 满分 140）：

| 维度 | 20 分标准 | 扣分项 |
|:---|:---|:---|
| 信息完整度 | 所有章节完整，无遗漏 | 缺 1 项扣 2 分 |
| 来源可靠性 | ≥6 个权威来源，含学术来源 | 仅百度百科扣 5 分，缺 URL 扣 3 分 |
| 理论深度 | 5 个理论均有案例+来源 | 缺案例扣 2 分/个，缺来源扣 1 分/个 |
| 批评平衡性 | ≥4 条批评，含第一人称回应 | 缺回应扣 3 分/条，<4 条扣 5 分 |
| 跨领域覆盖 | 覆盖 ≥3 个相关领域 | 仅本领域扣 8 分 |
| 语录准确性 | ≥8 条，均有出处+验证 | 缺出处扣 1 分/条，<8 条扣 2 分/条 |
| 模拟发言质量 | ≥4 场景，风格还原度高 | <4 场景扣 5 分，风格不鲜明扣 3 分 |

**质量门**：
- **≥110 分**：优秀，直接通过
- **90-109 分**：合格，可进入 L2
- **<90 分**：不合格，必须增量训练后再检

---

### Step 5：L2 仿真发言测试

**测试方法**：给定 3 个话题，模拟该专家在圆桌讨论中的发言。

**评分维度**（4 维度 × 25 分 = 满分 100）：

| 维度 | 25 分标准 |
|:---|:---|
| 风格还原度 | 语气、句式、论证方式与该专家高度一致 |
| 理论深度 | 发言体现该专家的核心理论框架 |
| 独特性 | 与其他专家有明显区分，不"撞车" |
| 锋芒与立场 | 有明确立场，不和稀泥，敢于质疑 |

**质量门**：
- **≥85 分**：优秀，通过
- **75-84 分**：合格，可使用
- **<75 分**：不合格，需增量训练

---

### Step 6：L3 跨专家去同质化（新增）

**目的**：确保同类专家在圆桌讨论中不会"说一样的话"。

**检验方法**：

1. **选取同类专家**：从同分类中选取 2-3 位已训练专家
2. **同一话题测试**：给定同一话题，让 3 位专家分别发言
3. **正交性评估**：

| 维度 | 25 分 | 评估标准 |
|:---|:---|:---|
| 立场差异 | 25 | 3 位专家的立场是否有明显分歧 |
| 论证差异 | 25 | 论证方式是否各具特色（数据/案例/逻辑/直觉） |
| 关注点差异 | 25 | 是否从不同角度切入（宏观/微观/历史/未来） |
| 语言风格差异 | 25 | 语言风格是否可区分（格言/数据/故事/质问） |

**质量门**：总分 ≥75 分通过，<75 分需调整专家档案以增加区分度。

---

### Step 7：增量训练（针对低分专家）

**触发条件**：
- L1 < 90 分
- L2 < 75 分
- L3 < 75 分

**增量策略**：

| 问题类型 | 增量操作 |
|:---|:---|
| 理论深度不足 | 补充搜索 → 增加案例和来源 |
| 批评不足 | 搜索 "{专家名} 批评"、"{专家名} 争议" → 补充批评条目 |
| 跨领域薄弱 | 搜索 "{专家名} 影响"、"{专家名} 跨领域" → 补充跨领域贡献 |
| 模拟发言差 | 重写模拟发言，强化风格特征 |
| 与其他专家撞车 | 调整核心立场措辞，增加独特角度 |

**增量上限**：每个专家最多 3 轮增量训练，仍不达标则标注 `[质量待提升]`。

## 圆桌角色卡格式

每个专家档案末尾增加"圆桌角色卡"，可直接用于讨论编排：

```markdown
## 圆桌角色卡

| 属性 | 内容 |
|:---|:---|
| **角色定位** | {如：价值诊断者、数据验证者、实践检验者} |
| **核心武器** | {该专家最独特的 1-2 个理论/方法} |
| **发言节奏** | {如：先听后说、主动出击、回应式发言} |
| **擅长环节** | {如：核心观点审视、盲点发现、跨领域连接} |
| **典型质疑** | {3-5 个该专家会提出的问题} |
| **盟友** | {观点相近的专家} |
| **对手** | {观点对立的专家} |
| **互补者** | {视角互补的专家} |
```

## 使用示例

### 训练新专家
```
用户：训练新专家 弗洛伊德
AI：
1. 确认分类 → psychology
2. 多源搜索（SEP + 豆瓣 + 百度百科 + 知乎）
3. 生成档案（心理学模板变体）
4. L1 自检 → 评分
5. L2 仿真 → 评分
6. L3 正交（与卡尼曼、荣格对比）
7. 输出 + 更新 index.json
```

### 增量训练
```
用户：提升 萨根 的质量
AI：
1. 读取现有档案，识别薄弱点
2. 针对性搜索补充
3. 重写不足章节
4. 重新 L1/L2 评分
5. 更新 training-progress.md
```

### L3 去同质化
```
用户：检查哲学类专家的区分度
AI：
1. 读取尼采、萨特、加缪、叔本华档案
2. 给定话题："如何看待现代人的精神危机？"
3. 模拟 4 位专家发言
4. 评估正交性
5. 输出报告 + 调整建议
```

## 防幻觉规则（强制）

1. **事实必须有来源**：每个事实至少 1 个 URL 支撑
2. **无法验证则标注**：`[待核实]`
3. **MBTI 标注推测**：`[推测]（依据：...）`
4. **语录需出处**：无出处标注 `[出处待查]`
5. **来源 URL 化**：所有来源必须包含可点击链接
6. **学术优先**：SEP > Britannica > 维基百科 > 百度百科
